protocolo HTTP e como ele é usado na Web

HTTP(Hypertext Transfer Protocol) é um protocolo que permite a obtenção de recursos, como documentos HTML. É a base de qualquer troca de dados na Web e um protocolo cliente-servidor , o que significa que as requisições são iniciadas pelo destinatário, geralmente um navegador da Web

  1. papel do HTTP na comunicação entre navegador e servidor

    O http atua na troca de dados entre o cliente e o servidor , por meio de um ciclo infinito de requisições por parte do cliente e respostas por meio do servidor

  2. oque é cliente, servidor, requisição e resposta

    • Cliente

      Quem inicia a comunicação

    • Servidor

      Recebe e processa a requisição do cliente

    • Requisição(Request)

      pedido do usuário , contendo uma série de dados que serão processados pelo servidor

    • Resposta(Response)

      Resposta da requisição do usuário , pode conter as informações necessárias ou algum código de erro

  3. Get e Post

    Get

    Usado quando o cliente deseja obter algum recurso do servidor,

    Post

    Usado quando o cliente deseja enviar algum recurso para serem processados no servidor

  5. Código de status

    1. Status 200

      Indica que a solicitação foi recebida pelo servidor , ou seja , que a mesma teve sucesso em se comunicar com o servidor

    2. Status 301

      Indica que uma página acabou sendo movida para outro endereço http

    3. Status 400

      indica que a solicitação possui algum erro e por isso não teve sucesso ao tentar se comunicar com o servidor

    4. Status 404

      Indica que a página acabou não sendo encontrada por meio do endereço solicitado pelo cliente

    5. Status 500

      Indica um erro que veio do próprio servidor
